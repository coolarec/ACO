# TSP 可视化教学项目设计

## 目标

使用 Vue 脚手架搭建一个单页教学项目，介绍并演示使用状压 DP 与蚁群算法求解 TSP 问题。页面左侧为配置与过程说明，右侧为算法可视化图。项目强调教学展示、参数可调、过程可观察，以及最终最短路结果的恢复与展示。

## 页面结构

页面采用左右分栏布局。

- 左侧为控制与说明面板
- 右侧为算法图可视化区域

左侧面板从上到下分为四个区域：

1. 公共配置区
- 点数 `n`
- 生成竞赛图按钮
- 刷新边权按钮
- 超时停止时间设置
- 当前最短路结果展示

2. 图数据概览区
- 当前点数量
- 点编号提示
- 当前图的边权统计信息

3. 状压 DP 展示区
- 算法介绍
- 运行按钮
- 停止按钮
- 当前过程模式提示
- 当前状态 `sta`
- 当前终点 `ed`
- 当前最优值
- 当前前驱节点
- 最终最短路展示

4. 蚁群算法展示区
- 算法介绍
- 运行按钮
- 停止按钮
- 信息素权重 `alpha`
- 可见度权重 `beta`
- 蚂蚁数量
- 当前迭代次数
- 当前轮最优路径
- 全局最优路径

## 可视化设计

右侧采用算法可视化风格的二维散点图，不使用真实地图底图。

图中应展示：

- 所有节点
- 节点之间的背景连线
- 当前算法关注的边
- 当前最优路径
- 最终最短路

不同算法的额外表现：

- 状压 DP 模式下，高亮当前状态涉及的终点节点和转移边
- 蚁群算法模式下，用边的透明度或粗细表示信息素强弱

## 图数据规则

### 点位生成

- 生成图时随机生成 `n` 个点的二维坐标
- 点位一旦生成，在刷新边权时保持不变

### 边权生成

- 任意两点之间存在边
- 边权为正整数
- 刷新边权按钮仅重置边权，不改变点位

## 状压 DP 设计

### 适用范围

状压 DP 仅适合较小规模点数。界面需自动根据点数切换过程展示粒度。

### 展示模式

- 小规模点数时，展示完整状态转移过程
- 点数较大时，自动切换为关键状态摘要模式

### 过程内容

每一步至少展示：

- 当前子集状态 `sta`
- 当前结束点 `ed`
- 候选前驱
- 更新后的 `dp[sta][ed]`
- 对应前驱记录

### 结果恢复

- 需要维护前驱数组或等价结构
- 算法结束后恢复完整最短回路
- 将最短路径与路径总长度显示在左侧与右侧

## 蚁群算法设计

### 可调参数

- 信息素权重 `alpha`
- 可见度权重 `beta`
- 蚂蚁数量

### 过程内容

每轮展示：

- 当前迭代次数
- 每轮最优路径长度
- 全局最优路径长度
- 信息素更新后的效果

### 结果展示

- 支持手动停止
- 支持超时自动停止
- 停止时保留当前全局最优路径作为结果展示

## 停止与超时

两种算法都应支持：

- 用户手动停止
- 超时自动停止

停止后应：

- 立即结束后续迭代或播放
- 保留当前已找到的最好结果
- 更新界面状态，允许重新运行

## 技术结构

项目基于 Vue 3 + Vite。

建议目录结构如下：

- `src/components/`
- `src/algorithms/`
- `src/composables/`
- `src/types/`
- `src/utils/`

其中算法相关逻辑必须单独放入 `src/algorithms/` 文件夹，至少包括：

- `src/algorithms/dpTsp.ts`
- `src/algorithms/acoTsp.ts`

界面组件建议包括：

- `src/components/ControlPanel.vue`
- `src/components/GraphCanvas.vue`
- `src/components/DpPanel.vue`
- `src/components/AcoPanel.vue`

辅助逻辑建议包括：

- `src/composables/useGraph.ts`
- `src/types/tsp.ts`

## 状态管理

- 图数据为全局共享状态
- 状压 DP 与蚁群算法共享同一份点和边权数据
- 两种算法的运行过程状态分别维护，避免互相覆盖
- 当前可视化高亮状态由顶层统一协调

## 交互原则

- 左侧为单页连续介绍，不使用标签页切换
- 用户先设置公共图数据，再分别体验两种算法
- 任一算法运行时，都能在右侧同步看到过程反馈
- 界面优先强调教学可读性，而不是追求大规模性能

## 测试与验证

至少验证以下内容：

- 能成功生成指定点数的图
- 刷新边权后点位不变
- 状压 DP 能在小规模图上恢复有效最短回路
- 大规模时状压 DP 自动切换为摘要展示
- 蚁群算法能根据参数变化得到不同搜索过程
- 手动停止与超时停止都能正确中断
- 右侧图能正确高亮当前路径与最终最短路

## 非目标

- 不接入真实地图服务
- 不实现后端
- 不追求大规模 TSP 求解性能
- 不要求两种算法同时并行求解
